Coding

Algorithm Making

Pair up and architect a Java algorithm from scratch — 1.5h to draft on paper, 2.5h to implement and submit.

Language: Java
Team Size: 2 members
Max Teams: 4 per category
Categories: Junior (9–10) & Senior (11–12)
Overview

Teams of 2 design and implement an algorithm in Java. The problem is revealed on the day of the event. The event runs in two back-to-back sessions.

Session Structure
Session 1 — 1.5 Hours
Draft Phase
Participants discuss the problem and write a rough draft of their algorithm on paper. No computers during this phase.
Session 2 — 2.5 Hours
Coding Phase
Participants implement their drafted algorithm on a computer and submit the final code to the event head.
Judging Criteria
  • Efficiency — how optimally the algorithm solves the problem.
  • Functionality — correctness of the output.
  • Optimization — memory and time complexity.
